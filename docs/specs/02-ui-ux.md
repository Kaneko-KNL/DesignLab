# UI/UX 仕様書

## 1. 画面レイアウト
VS CodeなどのIDEにインスパイアされた3ペイン構成を採用します。

### 全体構成
- **Header (上部)**: ロゴ、プロジェクト名、設定、ユーザープロファイル。
- **Sidebar (左)**: ナビゲーション（新規作成、エクスプローラー、学習、ドライブ）。
- **Workspace (中央)**: メイン作業領域。プレビューキャンバス。
- **Design Panel (右)**: デザインパラメータ設定エリア。

## 2. コンポーネント詳細

### Header
- 高さ: 56px
- 機能:
  - アプリケーションタイトルの表示。
  - 言語切り替え（JA/EN）。
  - 設定メニューへのアクセス。

### Sidebar
- 幅: 64px (アイコンのみの細いバー)
- 機能:
  - **New**: 新しいデザインプロジェクトを作成。
  - **Explorer**: 保存されたデザインファイルの一覧表示。
  - **Learn**: 色彩理論などの学習コンテンツ。
  - **Drive**: Google Drive連携（将来機能）。

### Workspace
- 機能:
  - **Tabs**: プレビューのコンテキスト切り替え（LP用、ブログ用、アプリ用など）。
  - **Preview Canvas**: 実際のUIコンポーネント（ボタン、カード、フォームなど）を表示する領域。背景色やフォントは設定をリアルタイムに反映。
  - **Action Bar**: ファイル名表示、「保存」、「プロンプト化」ボタン。

### Design Panel
- 幅: 300px
- 機能:
  - **Design Type**: プリセット選択（AI風、ガラス、和風など）。
  - **Color Palette**:
    - ランダマイザー（サイコロアイコン）: 理論に基づいた配色をランダム生成。
    - 個別カラー設定: Primary, Accent, Backgroundなどの色指定。
  - **Typography**: フォント選び（初期バージョンではシンプルに）。
  - **Shape**: 角丸（Radius）や影（Shadow）の強さ設定。

## 3. インタラクション
- **ホバー効果**: すべてのインタラクティブ要素に適切なホバー状態を設定し、「生きている」感覚を与える。
- **トランジション**: 色やレイアウトの変更は、0.2s〜0.3sの滑らかなトランジションで変化させる。
- **フィードバック**: 保存完了時やプロンプトコピー時には、トースト通知やアイコンの変化でユーザーに成功を伝える。
